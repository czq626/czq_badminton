# 进度记录

## 当前状态

项目已完成一个可以试玩、可以自检、适合提交到 GitHub 的第一版。

## 已完成

- 横版羽毛球场景和 16:9 Canvas 画布。
- 蓝队和红队火柴人绘制。
- 羽毛球飞行、重力、反弹、过网和落地判定。
- 挥拍、跳跃、捞球、扣杀、短球、发球等基础操作。
- 单人 AI 对战。
- 本地双人键盘对战。
- 独立开始界面，支持单人挑战、本地双人和房间模式选择。
- 房间模式改为房间列表主流程，支持刷新、创建房间和加入等待中的房间。
- 可选 WebSocket 房间服务支持跨设备创建/加入、房间心跳、离开和比赛消息转发。
- WebSocket 服务不可用时，房间模式回退为本机同源多标签房间列表。
- 加入方联机画面加入快照缓动、回合重置硬同步、远端玩家外推、HUD 降频和本地音效恢复。
- 单 HTML 打包脚本可生成内联 CSS、JS 和图片资源的 `dist/game.html`。
- 移动端触屏控制。
- 7 分和 11 分胜利目标。
- 先到目标分且领先 2 分获胜。
- 休闲、普通、高手三档 AI 难度。
- 标准和趣味两种球路风格。
- READY / GO 回合开始节奏。
- 比分、模式、发球方、难度、胜利目标等 HUD 信息。
- 落点辅助开关。
- 合成音效和静音开关。
- 菜单吸引态演示。
- 粒子、尘土、扣杀轨迹、胜利效果等视觉反馈。
- 挥拍时机和击球点会影响球路，包含甜点击球、贴网小球、平抽和高远球差异。
- 扣杀、捞球、平抽和网前小球进一步区分：地面强动作会高挑救球，空中高点进攻会向下压，低点进攻退化为平抽，短球更贴网。
- 捞球加入上挑反馈，扣杀加入更强命中反馈和短暂回位代价，降低无脑连续扣杀收益。
- AI 会按难度选择回高远球、平抽、网前小球或扣杀，站位和反应更有差异。
- AI 会根据玩家站位、长回合和被动球选择解围、压后场、拉网前或扣杀。
- 起跳、挥拍、捞球和扣杀加入短暂输入缓存，触屏挥拍按钮支持下滑短球。
- 发球加入标准高远发球、短发偷网前和快速长发差异。
- 长回合加入相持拍数、热度里程碑和更强击球反馈。
- 程序化美术升级：球场层次、场地纹理、球网细节、队伍色角色描边、羽毛球尾迹、HUD/菜单和触屏按钮视觉优化。
- AIART 像素风美术接入：生成并使用像素球馆背景、羽毛球透明精灵和扣杀爆闪素材，保留程序化绘制兜底。
- 开始界面和比赛 HUD 进一步正式化，使用生成球场素材做第一屏视觉预览。
- `localStorage` 和 cookie 兜底的本地偏好保存。
- `verify-game.js` 自检脚本。

## 当前验证结果

最近一次执行：

```bash
node verify-game.js
```

结果：90 项检查全部通过。

## 已知不足

- 球路和角色动作还没有逐帧对照原始游戏校准。
- AI 已加入站位战术，但还可以继续强化连续诱导、假动作和更拟人的失误。
- 火柴人动作已有挥拍残影和队伍色强化，但仍缺少更细腻的挥拍前摇和击球后摇。
- 移动端控制可用并加入基础容错，但按钮布局和更完整的滑动手势还可以继续打磨。
- 跨设备房间列表依赖外部 WebSocket 房间服务；单 HTML 产物只包含前端和客户端。
- 房间服务目前是内存版，重启后房间会清空，尚未做断线重连和持久化。
- 暂无自动化浏览器截图回归。
- 暂未配置 GitHub Pages 发布。

## 维护备注

- 后续改动玩法规则、操作方式、模式、视觉反馈或发布流程时，同步检查并更新 `README.md`、`CHANGELOG.md`、`docs/PROGRESS.md` 和 `docs/ITERATION_PLAN.md` 中相关内容。

## 发布基线

第一版适合作为 `v0.1.0` 或 initial commit：

- 可以本地运行。
- 可以完成完整比赛。
- 可以通过自检。
- 代码结构简单，便于后续继续调参和拆分。
