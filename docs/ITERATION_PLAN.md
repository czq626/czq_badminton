# 迭代计划

## v0.1.0 GitHub 基线

- 初始化 Git 仓库。
- 添加 `.gitignore`，避免提交 `.env`、`.DS_Store`、`.codex/`。
- 提交当前可玩版本。
- 补齐方向、进度和迭代文档。
- 后续绑定 GitHub 远程仓库并推送。

## v0.2.0 手感优化

- 已阶段性完成：调整击球窗口、扣杀速度、回合热度和长回合反馈。
- 拆出玩法参数表，降低调参成本。
- 已阶段性完成：增加不同击球点对球路的影响。
- 已阶段性完成：优化短球、高远球、短发和快速长发的可控性。

## v0.3.0 AI 优化

- 为 AI 加入更自然的反应延迟。
- 不同难度使用不同预判精度、移动速度和击球选择。
- 已阶段性完成：增加主动压网、后场防守、扣杀机会判断和按玩家站位变招。
- 在休闲难度保留合理失误率。

## v0.4.0 视觉和反馈

- 已阶段性完成：优化球场层次、场地纹理、球网细节、角色队伍色识别、球拍挥拍残影、羽毛球尾迹和 HUD/菜单视觉质感。
- 已阶段性完成：利用 AIART 生成像素风球馆、羽毛球和扣杀爆闪素材，并接入 Canvas 渲染链路。
- 继续优化火柴人跳跃、落地、挥拍前摇和击球后摇。
- 继续打磨击球瞬间的停顿、震动或速度线反馈。
- 继续调整比分、暂停、胜利态的信息层级。

## v0.5.0 移动端体验

- 优化触屏按钮尺寸和布局。
- 增加长按移动、滑动扣杀等更自然的输入方式。
- 检查不同手机比例下的遮挡和可读性。
- 降低移动端误触成本。

## v0.6.0 发布与质量

- 配置 GitHub Pages。
- 增加浏览器截图验证。
- 增加基础性能检查。
- 添加版本标签和发布说明。

## v0.7.0 联机房间

- 已阶段性完成：静态 HTML 下的同源房间码创建/加入流程。
- 已阶段性完成：房间模式改为房间列表主流程，玩家可以刷新、创建或加入房间。
- 已阶段性完成：接入可选 WebSocket 房间服务，支持跨设备自由创建和加入。
- 已阶段性完成：前端可打包为单 HTML，真实房间列表作为外部 WebSocket 增强。
- 继续增加更完整的断线重连、满员提示和好友邀请链接。

## 后续可能方向

- 更接近原版的皮肤和球场风格。
- 练习模式。
- 自定义比分、风力、球速和 AI 强度。
- 简单关卡或挑战模式。
- PWA 离线安装。

## 维护约定

- 每次更新玩法规则、操作方式、模式、视觉反馈或发布流程后，同步检查 `README.md`、`CHANGELOG.md`、`docs/PROGRESS.md` 和本迭代计划，避免代码和文档状态不一致。
