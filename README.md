# 一拍定胜负

一个零依赖 Canvas 火柴人羽毛球小游戏，复刻童年网页游戏的核心手感：横版球场、火柴人挥拍、跳跃扣杀、网前反弹、AI 对战、本地双人和房间列表联机对战。

## 运行

直接在浏览器打开 `index.html` 即可。

GitHub Pages 发布后，试玩地址通常为：

```text
https://czq626.github.io/czq_badminton/
```

也可以在当前目录启动一个静态服务：

```bash
python3 -m http.server 5173
```

然后访问 `http://localhost:5173`。

## 自检

```bash
node verify-game.js
```

自检会确认页面结构、关键按钮、Canvas 尺寸、样式/脚本引用和 `game.js` 语法。

## 房间服务

单人和本地双人仍然可以直接打开 `index.html` 游玩。跨设备房间列表需要单独启动 WebSocket 房间服务：

```bash
node room-server.js
```

默认监听 `ws://localhost:8787`。开始界面的“房间模式”会显示房间列表，玩家可以刷新、创建房间或加入等待中的房间。房间服务不可用时，会回退为本机同源多标签房间列表，方便离线测试。

## 单 HTML 打包

传播前端时可以生成一个内联 CSS、JS 和图片资源的单文件版本：

```bash
node build-single-html.js
```

产物位于 `dist/game.html`。真实跨设备联机仍需要外部 WebSocket 房间服务；单文件 HTML 内只包含游戏前端和 WebSocket 客户端。

## 操作

蓝队：

- `A` / `D`：移动
- `W`：跳跃
- `J`：挥拍/发球
- `K`：地面捞球 / 空中扣杀 / 快速长发
- `S`：网前小球

红队：

- 右侧默认由 AI 控制
- 点击顶部“切换”进入双人模式
- `←` / `→`：移动
- `↑`：跳跃
- `1`：挥拍/发球
- `2`：地面捞球 / 空中扣杀 / 快速长发
- `↓`：网前小球

通用：

- `P`：暂停/继续
- 顶部“切换”：单人 AI / 本地双人
- 顶部“辅助 / 硬核”：显示或隐藏落点辅助
- 顶部“音效 / 静音”：打开或关闭合成击球音效
- 起跳、挥拍、捞球和扣杀有短暂输入缓存，提前一点点按也能接上动作
- 发球时按短球键配合挥拍可偷发网前，扣杀键发球会打更快的长发
- 独立开始界面“单人挑战 / 本地双人 / 房间模式”：开局前选择 AI、本地或联机模式
- 菜单“休闲 / 普通 / 高手”：调整 AI 难度
- 菜单“7分 / 11分”：选择胜利目标分
- 菜单“标准 / 趣味”：选择普通球路或轻微风偏、弹性更强的趣味球路
- 房间模式使用房间列表：刷新、创建房间或加入等待中的房间
- 跨设备联机通过可选 WebSocket 房间服务实现；纯静态 HTML 下回退为本机同源多标签测试
- 房主控制蓝队和比赛同步，加入方控制红队
- 房间模式下两台电脑都使用蓝队键位操作：`A` / `D` 移动，`W` 跳跃，`J` 挥拍，`K` 地面捞球/空中扣杀，`S` 短球
- 手机宽度下会显示蓝队和红队两套触屏按钮，含移动、跳跃、挥拍、捞/扣和短球
- 手机触屏挥拍按钮向下滑可打短球
- 菜单和顶部偏好会保存在本机浏览器，下次打开自动恢复

默认 7 分制，先到目标分且领先 2 分获胜。
每回合开始会有短暂 `READY / GO` 提示，`GO` 后即可发球。
挥拍时机和击球点会影响球路；甜点击球、贴网小球、地面捞球和扣杀会有更明确反馈，扣杀后会有短暂回位代价。
长回合会累计相持拍数和热度提示，AI 会根据玩家站位选择压后场、拉网前、解围或扣杀。
